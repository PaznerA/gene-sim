# ADR-015 — More heritable traits under selection via the existing climate/soil seams (roadmap Phase T)

> **DRAFT — generated by the `phase-t-traits-design` workflow (scheduled run). DESIGN PROPOSAL ONLY — awaiting human sign-off. No code written.**

## Context

Phase T extends selection beyond the three shipped heritable traits (Genotype, DroughtTol, ThermalTol) by adding more per-individual traits coupled to environment signals the core already computes but does not yet consume — chiefly the climate `insolation` and `day_length` channels (climate.rs ClimateSample), which selection currently ignores entirely. The template is ADR-012 E3 (ThermalTol): a per-individual f64 component, a spawn draw appended in a FIXED order (lib.rs:384-387 currently `g0, energy, drought, thermal`), a `ClimateModifier`/`EnvironmentModifier` impl that scales fitness by ENVIRONMENT EXTREMITY so the temperate/default world stays selection-NEUTRAL, the component folded into `hash_world`'s id-sorted rows (lib.rs:659-705), and exactly ONE deliberate re-pin of the literal at lib.rs:739 (`0x9fad_2c9f_d298_f73a`).

Verified against the code:
- Spawn draws live in `reset_with_env` (lib.rs:384-387); a new trait appends one `unit_f64(rng.next_u64())` AFTER thermal, so every existing per-trait stream stays byte-identical.
- Selection (lib.rs:218-288) ALREADY fetches the global `clim_sample` (line 235) and ALREADY samples `local_soil` per parent (line 254). A climate-channel trait adds ZERO field lookups and ZERO loop draws — it just multiplies a second `fitness_factor()` into the weight and threads one more inherited f64 through the parents tuple / offspring tuple / `by_id` BTreeMap.
- `default_env_is_neutral` (climate.rs:178) asserts `day_length == 0.5` and `temperature == 0.5` to 1e-9 at default. Insolation at default = `day_length*(1-0.35*|lat_norm|)` = `0.5*(1-0)` = 0.5 EXACTLY (lat_norm=0). So an extremity gate `2*|signal-0.5|` is EXACTLY 0 at default for insolation, day_length, AND temperature — the modifier collapses to a flat 1.0 and the re-pin captures ONLY structure (one extra spawn draw + one extra hash column), with no behavioural mixing. This exact-neutrality holds for climate-channel traits but NOT for soil-channel traits: soil moisture/nutrients/pH are seed-derived per-cell fields with no 0.5-centering guarantee, so a soil-coupled extremity gate is nonzero on most pinned-run cells and its re-pin mixes structure + behaviour (the situation DroughtTol shipped under — acceptable but inherently messier and lower-priority for Phase T's clean-add discipline).

The candidate set was scored on (gameplay, determinism cheapness, seam fit). The clear winners are the climate-channel match-band clones (PhotoEfficiency, DayLengthGermination) plus the dual-channel PhotoThermalEfficiency, all of which are exact-neutral-at-default, reuse the already-fetched `clim_sample`, need no new module, and are pure +,-,*,abs,clamp (no transcendentals — invariant #3). Soil traits, the shared-temperature-channel MetabolicRate, the new-seam GrowthInvestment, and the deferred-payoff predation/relations traits (PredationResistance, ToxinProduction, Allelopathy) are explicitly out of scope for this ADR.


## Decision

Ship the top climate-channel traits, each as its own gated slice with its own single deliberate re-pin, in priority order. The recommended portfolio is 2 traits (one per currently-unconsumed climate channel — insolation and day_length), with a 3rd slice (PhotoThermalEfficiency) as an optional richer dual-channel follow-up that subsumes the insolation pick.

Mechanically, every Phase-T slice follows the ADR-012 E3 template EXACTLY: (1) add a per-individual f64 component; (2) append exactly ONE `unit_f64(rng.next_u64())` spawn draw in `reset_with_env` AFTER the prior trait in the fixed order; (3) add a sibling `ClimateModifier` impl using the extremity-band shape `factor = 1.0 + 0.5*extremity*align` with `extremity = 2*|signal-0.5|` (so default is flat 1.0); (4) in `selection`, thread the new f64 through the parents tuple + offspring tuple + `by_id` BTreeMap and multiply the new `fitness_factor()` into the weight using the already-fetched `clim_sample` (no new lookup, no loop draw); (5) fold the new component's `to_bits()` into `hash_world`'s id-sorted rows; (6) re-pin `determinism_hash_is_pinned` ONCE, ledgering the literal transition in the test doc-comment chain + ADR-015 consequences, mirroring the ADR-011/E3 procedure.

Do NOT ship two same-shape match-band modifiers on the same channel in the same portfolio (DormancyThreshold == DayLengthGermination geometrically; pick one). Do NOT ship the soil-channel or new-seam traits under this ADR — they need either fragile-neutrality acceptance or sim-core surgery and belong to a later ADR.


## Chosen traits

- PhotoEfficiency (insolation match-band; first/cheapest, lights up the unconsumed insolation channel)
- DayLengthGermination (day_length match-band; orthogonal photoperiod axis gated by latitude/season)
- PhotoThermalEfficiency (optional dual-channel insolation-benefit + day_length-cost; richer, subsumes PhotoEfficiency — ship EITHER PhotoEfficiency OR PhotoThermalEfficiency on the insolation channel, not both)

## Consequences

Selection now consumes the insolation and day_length climate channels (previously dead), giving the lat/lon/season menu real selective teeth and adding a sun-vs-shade and a photoperiod strategy axis orthogonal to the existing temperature axis. Each shipped trait adds: one per-individual f64 component, one appended spawn draw (existing streams byte-identical), one inherited tuple field through selection (no extra loop draw), one extra inlined `fitness_factor()` call in the hot loop reusing the already-fetched `clim_sample`, and one extra `to_bits()` column in `hash_world`. The pinned `determinism_hash_is_pinned` literal MUST change exactly once per shipped trait, in the same commit, ledgered in the test's history chain and here.

Because every chosen trait's extremity gate is EXACTLY 0 at the default env (insolation/day_length/temperature all == 0.5), the re-pin for each is STRUCTURAL-ONLY: existing tests (selection_responds_to_a_trait, the soil/drought tests, climate_coupling_adapts_thermal_tolerance_to_temperature) stay green because the new factor multiplies through as a flat 1.0 in their default-or-temperature-only worlds. Each slice adds a directed adaptation test (mirroring climate_coupling_adapts_thermal_tolerance) under a NON-default extreme-insolation / high-latitude-solstice world to prove the new coupling shapes selection. Future per-cell climate variation must come off the disjoint CLIM_STREAM_BASE family (climate.rs:15), never SimRng. Perf cost is negligible (one inlined unit-struct call per parent, no new field fetch) — no ADR-005 hot-path re-baseline needed, unlike the rejected GrowthInvestment.


## Slices

### T-A — PhotoEfficiency: per-individual heritable photosynthetic efficiency in [0,1] coupled to climate insolation via a sibling InsolationMatchModifier (verbatim transplant of TemperatureMatchModifier onto the insolation channel). Lights up the currently-unconsumed insolation channel.

- **Touches:** `crates/sim-core/src/climate.rs (add InsolationMatchModifier impl of ClimateModifier on climate.insolation; same extremity-band math)`, `crates/sim-core/src/lib.rs (PhotoEfficiency component; 5th spawn draw after thermal in reset_with_env ~line 387; thread through selection parents/offspring tuple + by_id BTreeMap; multiply new factor into weight using existing clim_sample; fold to_bits() into hash_world rows; re-pin determinism_hash_is_pinned literal)`
- **Determinism:** 🔁 RE-PIN
- **Acceptance:** gate.sh green; same_seed_same_hash + determinism_hash_is_pinned pass with the ONE new literal (history chain updated); a new directed test shows mean PhotoEfficiency rises under a high-insolation world (low lat + summer) and falls under a low-insolation world, from the same neutral ~0.5 start; all pre-existing tests unchanged (default insolation=0.5 makes the modifier flat 1.0). Pure +,-,*,abs — no transcendentals.

### T-B — DayLengthGermination: per-individual heritable critical-day-length threshold in [0,1] coupled to climate day_length via a sibling DayLengthMatchModifier (same extremity-band shape on the day_length channel). Adds a photoperiod axis gated by latitude/season.

- **Touches:** `crates/sim-core/src/climate.rs (add DayLengthMatchModifier impl of ClimateModifier on climate.day_length)`, `crates/sim-core/src/lib.rs (DayLengthGermination component; appended spawn draw after the prior trait; thread through selection tuple + by_id; multiply factor into weight via existing clim_sample; fold to_bits() into hash_world; re-pin literal again — one re-pin for THIS slice)`
- **Determinism:** 🔁 RE-PIN
- **Acceptance:** gate.sh green; determinism_hash_is_pinned updated to the new literal (history chain extended); a directed test shows mean threshold tracks the world's day_length under a high-latitude solstice world (long-day and short-day worlds diverge) from a neutral start, and is inert at the equator/equinox (day_length=0.5 → flat 1.0). All prior tests unchanged. Pure +,-,*,abs.

### T-C — OPTIONAL / ALTERNATIVE to T-A: PhotoThermalEfficiency — per-individual heritable photosynthetic temperature-optimum in [0,1] coupling insolation (benefit) AND day_length (cost) in a single second ClimateModifier impl, gated by light_extremity=2*|insolation-0.5|. Richer dual-channel gamble; subsumes PhotoEfficiency, so ship EITHER T-A OR T-C on the insolation channel, not both.

- **Touches:** `crates/sim-core/src/climate.rs (add PhotoThermalModifier impl reading both climate.insolation and climate.day_length: net = insolation*eff - (1-day_length)*eff; factor = 1+0.5*light_extremity*clamp(net,-1,1))`, `crates/sim-core/src/lib.rs (PhotoThermalEfficiency component; appended spawn draw; thread through selection tuple + by_id; multiply factor into weight via existing clim_sample; fold to_bits() into hash_world; one re-pin)`
- **Determinism:** 🔁 RE-PIN
- **Acceptance:** gate.sh green; one new pinned literal; directed test shows the trait rises under a bright long-day world and falls under a dim short-day world, and is exactly neutral at default (light_extremity=2*|0.5-0.5|=0 AND day_length=0.5 → flat 1.0 — REVIEWER must confirm BOTH channels are 0.5 at default, the only extra audit vs T-A). All prior tests unchanged. Pure +,-,*,abs,clamp.

## Re-pin plan

One deliberate re-pin per shipped trait, each in its OWN commit (ADR-011 one-isolated-re-pin-per-slice rule), sequenced in slice order. (1) After T-A (PhotoEfficiency): re-pin `0x9fad_2c9f_d298_f73a` → new literal; ledger the transition in the determinism_hash_is_pinned doc-comment history chain (`…now 9fad…f73a after E3; now <new> after T-A PhotoEfficiency — 5th spawn draw + insolation modulator, structural-only since default insolation=0.5`). (2) After T-B (DayLengthGermination): re-pin the T-A literal → another new literal; extend the chain analogously. (3) If T-C ships INSTEAD of T-A (recommended path picks T-A+T-B OR T-C+T-B), its re-pin replaces T-A's. Each re-pin runs the pinned 50-gen/1000-org default config and updates the single literal at lib.rs:739; never loosen an existing assertion to absorb the change. Re-pin SURFACE per slice = {the appended spawn-draw line, the selection tuple/BTreeMap field, the new fitness_factor product line, the new hash_world column, the pinned literal} — exactly the ADR-012 E3 envelope, nothing wider.


## Invariant risks

- #3 Determinism — the load-bearing risk. Each trait appends a spawn draw and a hash column, forcing a re-pin; the procedure must be executed EXACTLY once per trait in-commit, with the literal change isolated from any behavioural change. The extremity gate must use ONLY +,-,*,abs,clamp,min/max — NO sin/cos/exp (insolation/day_length are pre-computed scalars, so no trig is needed even for the 'timing' DayLengthGermination). Any iteration must stay id-sorted / BTreeMap-ordered, never HashMap.
- #3 (subtle) — neutrality depends on default insolation==0.5 (and day_length==0.5). day_length/temperature are asserted to 1e-9 in default_env_is_neutral, but insolation is NOT currently asserted there. Although insolation=0.5 follows arithmetically from day_length=0.5 and lat_norm=0, T-A/T-C SHOULD add an insolation==0.5 assertion to default_env_is_neutral to lock the neutrality contract against future climate.rs changes.
- #5 Pluggable science — new modifiers MUST be added strictly as sibling impls behind the existing ClimateModifier trait (the seam), reusing the extremity-band shape, without touching selection's arithmetic beyond adding the component + draw + fold + one fitness_factor product. SeasonalDormancy was rejected precisely because it would require widening ClimateSample (sim-core surgery beyond the seam); none of the chosen traits touch the ClimateSample struct.
- #2 Genome/biology in core only — all genotype→phenotype/fitness logic stays in sim-core/climate.rs; godot/ remains read-only and computes none of it. The new per-individual components are core ECS state, optionally surfaced into snapshots read-only later.
- #4 Headless-first — every slice is fully tested headless (directed adaptation test + pinned hash) before any renderer/specimen-view work.
