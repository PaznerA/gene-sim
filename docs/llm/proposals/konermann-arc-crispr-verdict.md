## CRISPRa/CRISPRi edit mechanic (the ship-now win)

YES — this is the single best, ship-now takeaway. Today an edit is effectively binary: apply_edit (crispr/src/lib.rs:724) gates on PAM + on/off-target, then perturb_value (lib.rs:675) shifts one Numeric activity param toward 0 (knockout). Konermann's toolkit gives the abstraction "CRISPR edit -> CONTINUOUS effect on a gene's activity" and the published deterministic relationships to land it as graded modulation. Add an edit_kind/intent {Knockout, Knockdown, Activate} on Edit (mirroring the existing EditType split at lib.rs:25, so a tunable-effect verb is already in the model's grain). Dispatch in apply_edit step 4: target_activity = curve(kind, efficacy), then move the param toward it and clamp_into_domain, where efficacy is the on-target score the pipeline ALREADY computes (DefaultOnTargetScore, lib.rs:413). Mechanic: Knockout(Cas9 DSB) -> activity 0.0 (today's behavior preserved); Knockdown(CRISPRi/CasRx) -> activity = 1.0 - efficacy*depth, a GRADED partial repression in (0,1); Activate(CRISPRa/SAM) -> activity = 1.0 + efficacy*gain, ABOVE wild-type. This makes an edit on ptsG/pflB/pta/ldhA/gltA produce a graded trait shift (e.g. GlucoseUptake 0.4 instead of 0/1) before any FBA/Evo oracle work, with ZERO new dependency. CRITICAL grounding correction: for the E. coli core the in-core default must use the E.-coli-CORRECT dCas9-CRISPRi repression rule (seed-sequence/TSS-position, Nat Commun 2018), NOT Konermann's mammalian Cas13d/SAM models — re-implement the fact, never import the off-distribution mammalian weights. Knockdown is the clean ship-now subset (lands in (0,1), no clamp change). Activate is the one friction: it needs activity > 1.0, but express() clamps to [0,1] (gp.rs:168) and ecoli.json's activity domain is [0,1], so overexpression is silently erased today — lifting that ceiling is a TAXONOMY.md genome-model decision that moves the pinned determinism hash, so it is a deliberate human-signed change, NOT free.
