# Sci-based sim-game — feature roadmap + zoom-scopes architecture (DRAFT)

> **DRAFT — generated by the `sci-game-features-scopes-design` workflow (8 agents). DESIGN/RESEARCH ONLY — strategic step-back, no code. Grounds on the ADR-013 joule engine.**

## Vision

gene-sim is an evidence-based CRISPR ecosystem sim-game: a deterministic Rust core simulates a living world as one conserved joule economy (ADR-013 CHEMOSTAT-J), and the player intervenes as a genetic engineer at the species/region level — designing CRISPR edits, watching emergent ecology and population genetics unfold, and being held honest by the science (real ontologies, SLiM-validated pop-gen, calibrated against textbook results). The signature experience is a continuous ZOOM from a single biological cell of one specimen, up through its tissues, organs, and whole body on the map, out to regional relations and the full ecosystem — where every scope is a different read of the SAME genome-derived, joule-denominated state, not a separate art asset. Every load-bearing number is traceable to a cited source, and a future LLM-genome layer (a process-boundary service, never linked, never on the sim path) lets real genome-based traits at all scales enter only as frozen, schema-validated, integer-quantized data the deterministic core consumes.


## Zoom scopes (max-zoom cell → ecosystem)

### cell (max zoom) — An individual biological cell of ONE selected specimen — a cell of the organism, NOT today's 32x32 world grid-cell (the current 'cells' zoom is a misnomer; it shows ecosystem grid aggregates).

- **Data shown:** Per-cell-type i64 joule/biomass sub-account; a small u16 expression vector (which loci are ON in this cell type — chloroplast/root/meristem); local toxin/signal load sampled from the organism's compartment; at deepest detail, the molecular-product abundances (which proteins/enzymes this cell type makes).
- **Genome→trait mapping:** Scale 0 MOLECULAR + Scale 1 CELLULAR of the expression pipeline: Locus(DnaSequence + Parameters + GO molecular-function tag) -> product-abundance vector via a baked codon/k-mer LUT (the bases finally do work beyond CRISPR PAM-finding, no transcription sim, no transcendentals); products + GO cellular-component/part_of tags -> per-cell-TYPE state. Cell types are deterministic, ontology-derived (GO cellular_component) nodes.
- **Core vs renderer:** CORE computes a CellTypeProfile per specimen: an ordered list of cell types, each with an i64 J share apportioned from organism Biomass via fixed::apportion, plus its u16 expression vector. RENDERER draws a few representative cell glyphs (extend the L-system leaf primitive) colored by expression channels; intra-glyph layout is presentation-only jitter (legal under inv #2). Computed ON DEMAND for the inspected specimen only — off the per-tick hash.

### cell-cluster / tissue — A tissue — an ordered group of like cell types (epidermis, mesophyll, vascular bundle).

- **Data shown:** Aggregate J/biomass of the cluster, dominant expression program, a tissue-level derived trait (e.g. water-retention from the same budget channel that yields organism DroughtTol).
- **Genome→trait mapping:** Derived from the cellular layer by deterministic single-link grouping of the CellTypeProfile by ontology part_of parent (reuses the ADR-013 F6 guild-clustering pattern at a pinned integer threshold). Cluster J = sum of member-cell J, so conservation closes by construction.
- **Core vs renderer:** CORE: integer sums over an ordered Vec keyed by stable ontology id, no RNG. RENDERER: draws clustered cell glyphs as a labeled tissue patch.

### organ — An organ / plant part (root, stem, leaf, flower) — the level today's L-system gestures at but does not back with real state.

- **Data shown:** Per-organ J/biomass budget; organ-level trait coefficients (leaf reflectance/light-capture, root drought-uptake, flower fecundity); the allocation trade-off made spatial (J in the root is J not in the flower).
- **Genome→trait mapping:** Scale 2 ORGAN/TISSUE: cell-type mix -> organ trait coefficients, each tagged to a GO biological-process / Plant-Ontology anatomy term. The organism Biomass is apportioned across the ordered organ list by the genome's EXISTING Strategy.budget ([u16;5] permille summing to 1000, ADR-013 F2) — the budget simplex becomes the organ J split with zero new allocation math. This is textbook root:shoot:reproductive optimal-partitioning, made literal by conservation.
- **Core vs renderer:** CORE: organ split = split_budget(biomass, strategy.budget), already conserved and tested; organ traits are the middle layer of the expression pipeline. RENDERER: this is where _plant_params_from_traits is REPARENTED — L-system root/stem/leaf/flower sizes come from core-exported per-organ trait vectors instead of a GDScript heuristic, so morphology is evidence-keyed to the budget.

### specimen-in-map — ONE individual organism at a real world Position (lib.rs Position on WORLD_DIMS), selectable and zoom-into-able — the currently-missing identity link between the ecosystem and the anatomy scopes.

- **Data shown:** Whole-organism trait vector (today's 5 traits + future open traits), Energy/Biomass, species, age/lineage net-J, NichePoint, and a handle to drill into its organs/cells.
- **Genome→trait mapping:** Scale 3 ORGANISM: organ coefficients roll up into the existing Strategy budget + TrophicRole + affinity that CHEMOSTAT-J already consumes. The multi-scale pipeline's TOP stage IS the F2 budget output, so nothing downstream in the sim changes — the budget is just now DERIVED through 3 inner scales instead of a flat sum.
- **Core vs renderer:** CORE: per-organism Energy/Biomass/Position/NichePoint already exist; the only ADDITION is a per-organism RECORD STREAM in the snapshot (new GSS kind, keyed by OrgId, walked in OrgId order off the RNG/hash path) plus a stable handle the renderer passes back to request that organism's OrganProfile/CellProfile. RENDERER: draws the single L-system plant at its map cell + an inspect panel.

### region — A sub-area of the map — a cluster of world-cells / a guild's territory / an agent's operational area. This is the natural agent-action scope (inv #6: agency is species/region level).

- **Data shown:** Per-species density planes; the FlowMatrix relations (who-eats-whom, synergy/parasitism/predation — the Rel signed-A heatmap); guild membership; aggregate pool depletion.
- **Genome→trait mapping:** No new genome mapping — this is aggregation of organism strategies over a window, plus the emergent trophic FlowMatrix derived from species TrophicRole/affinity (which themselves came from Scale 3).
- **Core vs renderer:** CORE: all of this is ADR-013 F4/F6 output (FlowMatrix, per-species snapshot planes, guild-clustering channel), folded in (i,j) order. RENDERER: the existing data-layer overlay windowed + a relations sub-view (signed-A heatmap).

### ecosystem — The whole 32x32 world — the well-developed scope today.

- **Data shown:** Today's 6-7 channels (density/allele_freq/fitness/soil_moisture/soil_nutrients/soil_ph) + new conserved-J pool and chemical-diffusion channels (ADR-013 F1/F5).
- **Genome→trait mapping:** Scale 4 POPULATION: organism strategies aggregated over the live run; allele_freq is the population-genetics read of the genome across all organisms. This is the scope the SLiM oracle validates against.
- **Core vs renderer:** CORE: the existing GridSnapshot, extended GSS2->GSS3 for pools/chem, off the RNG/hash path. RENDERER: today's ecosystem grid view, unchanged in role.

## LLM-genome layer (process boundary, inv #1/#2)

A new process-boundary subprocess crate (crates/oracle-genome), modeled EXACTLY on crates/oracle-slim: std-only, shells out to an LLM at the boundary, depends on nothing GPL and is NEVER linked into the game binary (inv #1). It separates WHO COMPUTES from WHEN IT RUNS via a two-phase contract. OFFLINE COMPILE (non-deterministic, runs once per genome/ontology change): the subprocess takes the genome loci + their SO/GO OntologyTags + the pinned OntologyGraph and PROPOSES a multi-scale ExpressionTable — for each locus, which molecular products it makes (GO MF), which cell types express it (GO CC/part_of), which organs it builds, and how those roll up into the organism Strategy budget. The LLM's job is retrieval-and-mapping over NAMED real datasets, not free generation. Each proposal must clear three gates or be REJECTED: (a) validate against the TAXONOMY section-4 JSON schema; (b) be is_a/part_of-subclassed under an existing SO/GO/ENVO node; (c) carry a resolvable cited SourceId (a proposal with DerivationKind=LlmProposed but no resolvable citation is rejected, never silently admitted as an assumption). Admitted proposals are quantized to integers at the SINGLE audited chokepoint fixed::to_unit_u16 and written as a versioned, content-hashed, integer-only FROZEN ARTIFACT (like specimens.json but multi-scale and schema-admitted). ONLINE CONSUME (deterministic, in-sim): the core reads ONLY the frozen integer ExpressionTable — pure ordered integer math, byte-identical on every platform; the table's content-hash is folded into the run's determinism hash (a different LLM compilation is simply a different ledgered run). If the artifact is missing or stale, the core falls back to the in-core DEFAULT expression map (inv #5: pluggable, in-core default + boundary 'realistic' impl), so the sim NEVER blocks on the LLM and a run with no LLM artifact is still fully deterministic. The LLM's non-determinism is thus QUARANTINED to compile time; it never runs on the sim path, never touches the hash, never appears in GDScript (inv #2). This is the same freeze-then-consume contract the SLiM oracle already uses, and it is what lets every scope (molecular -> population) be 'real genome-based' without breaking determinism.


## Feature roadmap

### Tier 0 — Game spine (unblocks everything) — Make the deterministic simulator a game by giving runs config, goals, and provenance.

- Scenario spine: generalize seed.json + actions.ndjson into ONE hashable RON/JSON Scenario {master_seed, pinned versions, world dims, species roster+genomes, env/soil/climate params, edit budget, predicates} — the determinism artifact every other pillar serializes/diffs (= R4, reframed as foundation).
- Objective/predicate engine: core-side, RNG-free evaluator over the per-gen stats stream; typed predicates (allele_freq>=X by gen G, population in band, kill-switch self-exhausted, protected-locus untouched) carried as DATA in the Scenario; renderer reads a per-gen pass/fail/par vector. Hash-neutral; turns every sim feature into a goal.
- crates/provenance: Sourced<T> wrapper (value + SourceId + DerivationKind + admitted schema version) that derefs to value so it adds ZERO bytes to hash_world; checked-in data/evidence/sources.ron registry. The spine all evidence work hangs on.
- *depends on:* ADR-012 env params (done), actions.ndjson replay contract (done)

### Tier 1 — The CRISPR game loop (mostly exposing existing core) — Deepen the core verb (CRISPR design) and make the sim legible and shareable.

- CRISPR guide-design workbench: renderer panel surfacing the already-core-computed rust-bio PAM sites + on/off-target scores per candidate guide BEFORE commit; renderer requests, core applies (R5 pattern). Exposes existing crates/crispr science as the central game verb — zero new biology.
- Edit budget / cost-risk economy: each edit costs from the Scenario edit-budget, off-target hits add penalty, failed-edit perturbation has real cost — design becomes a tradeoff, not a free retry. Seeded/deterministic in core.
- In-game telemetry/analysis surface: first-class time-series + cross-run panels over the existing per-gen Parquet (population, allele-freq, trait distributions, joule pools post-F1, seed-divergence diff). Read-only renderer — the Cities-Skylines info-overlay identity.
- Sharing/replay/challenge/leaderboard: export/import a run as the tiny Scenario+actions file; deterministic run-hash badge; a challenge = Scenario minus solution; leaderboard keyed by outcome hash + edit budget used; verification = re-run + compare existing stats hash.
- *depends on:* Tier 0 (Scenario spine + predicate engine); crates/crispr (done), determinism gate (done)

### Tier 2 — Evidence mandate (the user's #1 priority; renderer-light, parallel) — Make 'evidence-based' a STRUCTURAL, gated property rather than an aspiration.

- check_provenance gate (wired into gate.sh): fail the build if any value reachable from the sim hot path is Heuristic/Assumption without an explicit DECISIONS.md allowlist entry; assert every used SourceId resolves in sources.ron. Makes 'defensible' a gate, not a judgment call.
- Wrap existing sim constants in Sourced<T>: retrofit SlimParams defaults, CasVariant PAM rows, ADR-013 budget/role tables with real citations; flag gp.rs weights as Assumption so the gate surfaces them. Determinism-neutral (metadata only).
- Pinned real ontology snapshots (SO/GO/ENVO/NCBI-tax): replace opaque u32 stubs with release-versioned OBO snapshots loaded as data through the TAXONOMY section-4 is_a admission gate, pinned in DECISIONS.md (inv #7); ENVO grounds Climate/Soil, NCBI-tax grounds SpeciesId.
- crates/calibration pop-gen + ecology gates: assert neutral runs match Wright-Fisher (1/2N fixation, heterozygosity decay) AND agree with the SLiM oracle within tolerance; assert CHEMOSTAT-J reproduces Tilman R*, competitive exclusion, Monod uptake, Lotka-Volterra regimes. Build is RED until the sim reproduces textbook results — the concrete meaning of 'evidence-based engine'.
- crates/oracle-crispr + Doench rank-corr golden test: subprocess (oracle-slim template) shelling out to Bioconductor crisprScore (Rule Set 2/Azimuth, DeepHF) and CFD/Crisflash for off-target; the 'realistic' tier of the inv-#5 score traits; golden test asserts the in-core scorer tracks published scores by Spearman correlation.
- Provenance surfaced in renderer INSPECT panel: core exports a Sourced source-string per inspected value; INSPECT shows the citation + evidence class (heuristic / SLiM-validated / LLM-frozen) next to the number. Read-only.
- *depends on:* crates/provenance (Tier 0); SLiM oracle (done); TAXONOMY section-4 gate

### Tier 3 — Multi-scale zoom scopes core data model (headline epic; STOP-THE-LINE, needs ADR + sign-off) — Turn the single-granularity snapshot into a scope-indexed read of one conserved-J, ontology-keyed economy spanning cell->ecosystem.

- OntologyTags-keyed multi-LEVEL expression pipeline in gp.rs: refactor the flat WeightedSumMap into Parameter->molecular product(GO MF)->cell-type vector(GO CC/part_of)->organ traits(Strategy budget)->organism traits->population, each level behind the GenotypePhenotypeMap trait (inv #5) with an open ontology-keyed TraitId(GoTermId,Scale) replacing the closed 5-variant enum. Prerequisite: the ADR-013 F2 flat-index fix. The data spine of every scope.
- Conserved-J sub-ledger: apportion an organism's Biomass/Energy across organ->tissue->cell compartments via fixed::apportion (organ split = the existing Strategy budget simplex); extend ledger_closes to sub-accounts; re-prove the i64 overflow margin. Internal anatomy becomes one more conserved layer of the SAME CHEMOSTAT-J economy, not a parallel system.
- On-demand per-specimen anatomy decomposition (OrganProfile/CellProfile): a read-only, off-hash expression (modeled on harness write_specimens / a separate GeneSimEnv) producing one specimen's organ/cluster/cell profiles only when the user inspects it — keeps inner scopes off the per-tick hot path (inv #6), adds no live re-pin.
- Per-organism record stream + GSS3 multi-scale snapshot kind + stable specimen handle: new read-only snapshot kinds keyed by OrgId (off RNG/hash path, magic-versioned GSS2->GSS3) providing the specimen scope and the click-to-drill identity link.
- Reparent L-system to core-exported organ trait vectors + cell/cluster glyph rendering: drive _plant_params_from_traits from core per-organ vectors; extend primitives to draw cell glyphs and tissue patches colored by core expression channels; per-scope inspect panels. Biology moves fully into the core.
- Scope ENUM + scope-state machine (renderer): replace the 3-bucket magnification + 2-view toggle with explicit {Cell,Cluster,Organ,Specimen,Region,Ecosystem}, each bound to one core stream, LOD-loading inner data only when zoomed in. Region scope = windowed ecosystem + FlowMatrix relations sub-view. Build LAST (renderer).
- *depends on:* ADR-013 F2 (ontology-keyed expression) + F4 (per-organism/per-species streams + FlowMatrix) + F6 (guild clustering); crates/provenance

### Tier 4 — LLM-genome layer (Stage 5; STOP-THE-LINE, needs ADR + sign-off) — Real genome-based traits at all scales, admitted only as frozen schema-validated cited integer data.

- Frozen multi-scale ExpressionTable artifact: versioned, content-hashed, integer-only per-locus (GoTermId, Scale, i64 weight) rows + ontology refs/citations; quantized via fixed::to_unit_u16; content-hash stamped into seed.json. The deterministic contract the layer rests on.
- crates/oracle-genome boundary subprocess (compile-only): oracle-slim template, never linked (inv #1); calls the LLM offline, validates proposals against the TAXONOMY section-4 schema + is_a/part_of admission + resolvable cited SourceId (or REJECT), quantizes to integers, writes the frozen ExpressionTable. Caller-derived seed for any sampling.
- Scale-indexed pipeline consumes the ExpressionTable with an in-core DEFAULT fallback (inv #5) so the sim runs with no LLM and stays fully deterministic; missing/stale artifact -> default map.
- *depends on:* Tier 3 multi-scale pipeline; Stage-5 ontology schema gate; pinned ontologies (Tier 2)

### Cross-cutting — Accessibility & onboarding — Make the dense info-game learnable and inclusive; ship incrementally with each UI tier.

- Colorblind-safe overlay palettes (data-layer shaders select channel by index, so palette is just a uniform).
- Keyboard-navigable scopes/inspect.
- A tutorial Scenario that teaches the CRISPR loop via the predicate engine.
- Glossary/ontology tooltips driven by GLOSSARY.md + the ontology surface.
- *depends on:* Tier 0 predicate engine (tutorial); data-layer shaders (done)

## Evidence-based strategy

Evidence-based becomes a STRUCTURAL property enforced by three build gates rather than a promise, and both subprocess oracles (SLiM today, LLM later) are constrained to feed only cited, schema-validated, integer-frozen data into the deterministic core. (1) PROVENANCE AS DATA: a crates/provenance Sourced<T> wrapper carries {value, SourceId, DerivationKind (Direct/FittedTo/Assumption/OracleFrozen), admitted schema version}; it derefs to value so the sim path reads only the i64/u16 inside and it adds zero bytes to hash_world (determinism-neutral). A checked-in data/evidence/sources.ron registry rows {id, kind:(PeerReviewed|Database|Ontology|Heuristic|LlmProposed), doi/accession, title, retrieved, license}. (2) PROVENANCE-REACHABILITY GATE (tools/check_provenance.sh in gate.sh): the build FAILS if any value reachable from the sim hot path is Heuristic/Assumption without an explicit DECISIONS.md allowlist entry, and every used SourceId must resolve — turning 'is this number defensible?' from a code-review judgment into a red/green gate. (3) REAL PINNED ONTOLOGIES, loaded as data and never linked, admitted via the TAXONOMY section-4 is_a gate, pinned by release IRI+date (inv #7): Sequence Ontology for locus kinds, Gene Ontology (MF/BP/CC — the three aspects map onto the three inner pipeline scales), ENVO for biome/climate/soil so an environment is a real biome term not a magic float, NCBI Taxonomy for SpeciesId; loci seeded from real Ensembl/Ensembl-Plants GFF3 + UniProt annotations so a 'growth_locus' traces to an actual gene model. (4) CALIBRATION-AS-A-GATE (crates/calibration, GATE_CALIBRATION=1, off the hashed run): the build is RED until the headless core reproduces textbook/published results within pinned tolerance — pop-gen vs Wright-Fisher diffusion (1/(2N) fixation, heterozygosity decay) AND cross-checked against the SLiM oracle via tskit/pyslim (disagreement = stop-the-line); ecology vs Tilman R* equilibrium, competitive-exclusion, Monod uptake, Lotka-Volterra/Rosenzweig-MacArthur for the CHEMOSTAT-J substrate; CRISPR vs Doench-2016 Spearman rank-correlation. (5) CRISPR SCORING GROUNDED IN PUBLISHED PREDICTORS behind the inv-#5 traits: in-core integer/table-driven defaults calibrated to track Rule Set 2/Azimuth and CFD, plus a crates/oracle-crispr subprocess shelling out to Bioconductor crisprScore / CFD / Crisflash as the 'realistic' tier; PAM rows wrapped in Sourced<T> with originating papers. (6) TRANSCENDENTAL math (diffusion expectations, Monod/Lotka-Volterra ODEs, predictor floats, LLM logits) is ALLOWED only at the boundary or in the calibration harness, compared against the sim within a tolerance band, never executed on the deterministic path; any boundary number that will enter the core is quantized once at the single audited chokepoint fixed::to_unit_u16, recorded with the producing run-hash (OracleFrozen), and thereafter is plain frozen data. The LLM is the biggest evidence risk (hallucinated biology) and is solved the same way: offline + frozen + admitted only with a resolvable citation, else rejected — its job is retrieval-and-mapping over named datasets, not free generation. Net: every load-bearing number is traceable to a source, calibrated against published reality, and bit-reproducible.


## ⚖️ Open questions for human

1. SEQUENCING: Tier 3 (multi-scale scopes) and Tier 4 (LLM layer) depend on ADR-013 F2 (ontology-keyed expression) and F4 (per-organism/per-species streams + FlowMatrix) landing first. Do you want to push ADR-013 to completion (F0-F7) before any scope work, or interleave the cheap evidence/game pillars (Tiers 0-2) in parallel while the engine matures?
2. INNER-SCOPE COST MODEL: the recommended design computes cell/cluster/organ scopes ON DEMAND for only the inspected specimen (modeled on harness write_specimens, off the hash, no live re-pin) rather than for every organism every tick. Do you accept that the inner anatomy is a render-driven decomposition of a SELECTED specimen (so it is never part of the live population hash), or do you want internal compartments folded into the per-tick hash for every organism (far more expensive, multiplies re-pins, must re-prove the i64 overflow margin)?
3. TRAIT SPACE BREAKING CHANGE: realizing real genome-based traits at all scales requires retiring the closed 5-variant Trait enum in gp.rs for an open ontology-keyed TraitId(GoTermId, Scale). This is a 🛑 invariant-touching refactor (inv #2/#3/#5) of the genotype->phenotype spine. Approve as a dedicated ADR with its own design workflow + human sign-off (not an in-loop slice)?
4. ONTOLOGY ACQUISITION: pinning real SO/GO/ENVO/NCBI-tax snapshots and seeding loci from Ensembl/UniProt means vendoring sizable release-versioned data and recording IRIs in DECISIONS.md (inv #7). Is that data footprint + licensing acceptable in-repo, and which species/genomes should the first real reference genomes be (which biome/taxon do you want the PoC grounded in)?
5. LLM PROVIDER + DETERMINISM CONTRACT: the LLM-genome layer is offline-compile-only and its output is frozen+content-hashed before the sim reads it. Which LLM/CLI do you intend at the boundary, and do you accept that a different model/version producing a different ExpressionTable is, by design, a DIFFERENT ledgered run (its content-hash is folded into the run hash)?
6. CALIBRATION TOLERANCES: making the build RED until the sim reproduces Wright-Fisher / Tilman R* / Lotka-Volterra / Doench results requires you to sign off on the tolerance bands (how close is 'agrees with the literature'). Who owns those numbers, and is in-core-vs-SLiM disagreement beyond tolerance a true stop-the-line halt?
7. GAME IDENTITY: is the target experience closer to a Cities-Skylines-style legible info-sandbox (telemetry + overlays + free experimentation) or a goal-driven challenge/campaign game (predicate-scored scenarios, leaderboards, edit budgets)? Tiers 0-1 build the spine for both, but it shapes which Tier-1 features get depth first.
8. AGENCY GRANULARITY AT INNER SCOPES (inv #6): the scopes RENDER a selected specimen's internal cells/organs, but the player still acts at the species/region level. Confirm there is no intended player action AT the cell/organ level (e.g. no per-organism surgery) — i.e. inner scopes are inspect-only, edits remain species/region-wide.
