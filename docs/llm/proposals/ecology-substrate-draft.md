## Decision

Adopt CHEMOSTAT: a single conserved fixed-point energy/mass currency (the "joule" J, i64 quanta) as the substrate spine. Every load-bearing quantity — per-cell resource pools, per-organism stores, biomass, trophic transfers, chemical concentrations, reproduction endowments — is denominated in J over a global LEDGER conserved exactly modulo exactly THREE named, audited taps: INFLUX (solar/light minted per tick), RESPIRATION/LOSS (maintenance + trophic-efficiency dissipation), and OVERFLOW (the explicit sink for any cap-saturation event — added per the adversarial fix so saturating_add can never silently destroy a quantum). "Fitness" is deleted as a stored input and re-emerges only as a MEASUREMENT (realized lineage net-J). The four pillars are four classes of J transfer over the one ledger; niche geometry and the trophic flow-matrix are emergent measurements layered on top.

This is the BOLDEST of the three integrated architectures that survived the pressure-test as buildable: of the two CHEMOSTAT variants, the "One Conserved Joule, Staged Re-Pins" variant got determinism_holds=true (the other got false for conflating "integers can't diverge" with "the simulation can't diverge"); FluxNet also held but its FlowMatrix-as-flow-network framing is grafted IN rather than competing. We graft FluxNet's explicit directed-flow framing for the emergent community matrix, and apply EVERY adversarial fix as binding contract (below).

We DELETE: selection() (lib.rs:218), fitness() (lib.rs:207), the [0.5,1.5] band + 0.05 floor, the no-op metabolism() (lib.rs:197), the 5-scalar WeightedSumMap, and unit_f64 from the sim path. We REPLACE with an explicit fixed-order pipeline: advance_tick → influx → diffuse+decay chem → emit chem → uptake/convert metabolism → trophic_transfer → pay-maintenance → reproduce_or_die → measure_relations. Population becomes a free variable; extinction is permitted and desired (supersedes ADR-005).


## Chosen architecture

CHEMOSTAT-J: one conserved i64 "joule" ledger, four operations, staged re-pins — grafting FluxNet's directed-flow community-matrix framing and hardened with all adversarial fixes.

DATA MODEL. type J = i64 (the unit IS the quantum; no float scale). Per-cell pools Vec<i64> per channel {light, free_nutrient, detritus} row-major (cell = y*W+x, WORLD_DIMS=SOIL_DIMS=32x32 reused). Per-cell ChemField Vec<i32> per signal channel + pre-allocated scratch buffer for double-buffered diffusion. Per-organism components, all integer, all hashed in OrgId order: OrgId(u32, monotonic NEVER-reused via NextOrgId), Position{x,y}, Species(SpeciesId u16), Energy(i64) [repurposed from the dead f64], Biomass(i64), plus a SPECIES-level cached Strategy (in SpeciesRegistry, not per-org, honoring inv #6). Strategy{ budget:[u16;5] summing EXACTLY to 1000 permille, role:TrophicRole{Autotroph,Heterotroph,Mixotroph,Decomposer}, affinity:[u16;C], derived physiology coefficients }. Heritable per-org standing variation is a single u16 NichePoint scalar (replaces Genotype/DroughtTol/ThermalTol; the latter two become DERIVED from budget, not free draws). Resources: Ledger{influx_total, respired_total, overflow_total:i64}, FlowMatrix (flat Vec<i64> length S*S, the emergent community matrix), SpeciesRegistry (ordered Vec, caches expressed Strategy), SimRng (single ChaCha8Rng, unchanged).

THE LEDGER. ledger_closes test asserts every tick: Σ(all pools + all chem + all organism Energy+Biomass) == initial_total + influx_total − respired_total − overflow_total. A semantic gate STRONGER than the bit-hash, with ALL three taps named (the third, overflow, added per the verdict's cap-saturation fix).

ORDERING (binding contract, per the #1 fix). NO CHEMOSTAT system reads dynamics-affecting state from Bevy Query iteration order. Every order-dependent pass first collects into a Vec sorted by (cell_index, SpeciesId, OrgId) — the collect-then-sort_unstable_by_key pattern selection() already uses at lib.rs:246 — then iterates that buffer, then writes back via ordered BTreeMap keyed by OrgId. Trophic transfer (which needs two &mut to different entities) uses an explicit EntityId index buffer + snapshot-of-prey-stores + ordered apply; the contention/visibility semantics (does attacker N see attacker N-1's debit this tick?) are PINNED as a design decision, not left to impl.

ARITHMETIC. Saturating ops EXCEPT on J-carrying ledger accounts, which use checked_add routing overflow to the named overflow tap. Every "divide" is a right-shift by a pinned power-of-two OR numerator*share/PINNED_DENOM with floor rounding and largest-remainder distribution, ties to the LOWEST index (channel→OrgId→SpeciesId). One shared, tested det-rounding module (fixed.rs) holds the apportionment + tie-break helper, reused by the budget simplex, diffusion remainders, and trophic division (per the "promote one canonical rounding module first" fix). Diffusion: 4-neighbour stencil, share=c>>diffuse_shift, remainder kept in-cell, reflecting boundary returns off-grid share to self → Σ-before==Σ-after asserted. Geometry: squared-L1/L2 in i64 (no sqrt); overlap is a BAKED const integer LUT triangle kernel (no Gaussian/exp; LUT contents are const array literals checked into source so cross-platform bytes are identical by construction). Trait-match: Σ(SCALE − |attack_d − defense_d|), multiply/add/abs only.

QUANTIZATION CONTRACT (authored here, not reused). UNIT_SCALE = 65535 (u16), q(x) = round(clamp(x,0,1)*65535); Q16 = i32 with implicit /65536 for signed coordinates. Pinned in a new fixed.rs module + its own sub-section of this ADR. as_unit_scalar migrates to an integer ratio.

GENOME→STRATEGY. New BudgetAllocationMap impl of (a widened) GenotypePhenotypeMap emits Strategy keyed off locus OntologyTags ids (NOT flat index — unblocks distinct per-species genomes). The trait gains an associated Output type (Phenotype for legacy, Strategy for CHEMOSTAT) per the GP-seam-break fix.

GRAFT FROM FluxNet. The FlowMatrix is framed as a directed flow network: A[i][j] = net J that physically flowed from species j into species i this generation (predation/grazing/parasitism positive into consumer, mutualism bidirectional) minus realized resource contention, accumulated in sorted (i,j) order. This is the emergent inversion of the Rel-draft's similarity-cosine.


## The edge (what's radical + why worth it)

What is genuinely radical: "fitness" stops existing as a number. There is no weight, no lottery, no fixed N, no [0.5,1.5] band, no floor. An organism lives or dies by whether a CONSERVED currency balances — it harvests J from a shared finite cell pool (so cell-mates genuinely contend and the second by OrgId gets the remainder), moves J out of prey into itself, spends J to poison neighbours through a diffusing field, and funds a child only from real surplus. Population size, carrying capacity, density-dependent crashes, trophic cascades, competitive exclusion, allelopathic warfare, kin cooperation, and EXTINCTION all emerge from one accounting identity rather than from tuned rules. Relations invert from a fabricated input coefficient to a measured output flow.

Why it is worth the cost: the substrate does triple duty. (1) It makes the four pillars ONE system (J transfers) instead of four bolt-ons that would each need their own determinism audit. (2) It makes determinism a property of the TYPE for the ARITHMETIC — retiring the f64 .to_bits() crutch (lib.rs:672) that today silently risks cross-platform divergence — which is the strongest determinism win available and the reason the big re-pin pays for itself. (3) The conservation ledger gives a SEMANTIC correctness invariant (ledger_closes) that catches modeling bugs the re-pinned bit-hash would otherwise silently bless. The honest caveat the verdicts forced in: integer arithmetic alone does NOT make the SIMULATION deterministic — iteration order, draw order, write visibility, and variable birth-draw-counts do, and those are the real engineering surface this ADR must specify rather than wave away. That is why the ordering contract, the explicit index buffers, the draw-count fold, and the multi-platform reproducibility harness are MANDATORY, not optional.


## Fixed-point determinism contract

SINGLE CURRENCY: type J = i64 quanta (1 J = smallest indivisible energy packet). Pools Vec<i64>, Energy(i64), Biomass(i64), chem i32 milli-units, ledger taps i64. 32x32 grid × pinned caps keeps totals far under i64::MAX; a worst-case-ceiling test (cells×max_cap + maxpop×max_store < i64::MAX with margin) is REQUIRED and asserts no individual transfer overflows before saturation.

ONE [0,1]-PROXY SCALE, AUTHORED HERE (no phantom reuse): UNIT_SCALE=65535, q(x)=round(clamp(x,0,1)*65535)→u16; Q16=i32 (/65536) for signed coords. Pinned in crates/sim-core/src/fixed.rs with const literals + its own ADR sub-section. genome ParamValue::Numeric f64 → integer; as_unit_scalar → integer ratio value*65535/(max-min) as u16 (the single former float-division chokepoint, genome/src/lib.rs:113, eliminated).

NO TRANSCENDENTALS, NO FLOAT, NO FLOAT-DIVISION on the per-tick/hash path: no sin/cos/exp/sqrt-of-float/pow-frac. Every divide is a right-shift by a pinned power-of-two, OR num*share/PINNED_DENOM with FLOOR rounding + largest-remainder leftover distribution, ties to LOWEST (channel→OrgId→SpeciesId). ALL rounding directions + remainder routing pinned in ONE table in this ADR and implemented in ONE shared tested fixed.rs helper (so ledger_closes cannot be satisfied by two compensating rounding bugs). Geometry LUTs are BAKED const array literals (never computed from f64 at startup). Budget simplex sums to EXACTLY 1000 via largest-remainder; re-closure point is PINNED as "mutate raw integer weights, THEN apportion" with a unit test that a 1-quantum mutation changes at most the mutated channel ± the single remainder recipient.

CONSERVATION: ledger_closes asserts Σ(pools+chem+Energy+Biomass) == initial + influx_total − respired_total − overflow_total EVERY tick (overflow is the named third tap closing the cap-saturation leak the verdict found). Diffusion asserts Σ-before==Σ-after. A skipped over-cap birth does NOT consume the parent's endowment J (pinned, else the allocator guard leaks).

RNG: single ChaCha8Rng, seeded once (lib.rs:368), never re-seeded; unit_f64 RETIRED from the sim path (bounded draws = next_u64()%range; fixed-point fraction = x>>48). Draws happen ONLY at births, in fixed (SpeciesId,OrgId) order, shape = D mutation draws then 1 dispersal draw. Draw COUNT becomes variable; to make order bugs fail FAST, the per-tick total next_u64 call count is FOLDED INTO THE HASH (per the verdict fix) — an off-by-one in birth enumeration then fails locally instead of drifting silently.

HASH: hash_world (lib.rs:652) drops EVERY .to_bits(); folds raw integers in fixed order — per-org (OrgId, Energy, Biomass, Strategy/NichePoint words, Position) by OrgId; per-cell pools+chem row-major; FlowMatrix in (i,j) order; the three ledger taps; the draw-count; one trailing SimRng word.

DETERMINISM PROOF OBLIGATION (binding): a multi-platform CI matrix (x86_64 + aarch64) runs same_seed_same_hash on the integer pipeline — the "determinism is a property of the type" thesis must be PROVEN on two ISAs, not asserted. An archetype-reorder test (re-spawn in shuffled archetype, assert identical hash + identical FlowMatrix) guards the ordering contract off-hash.

OFF-STREAM SEED REGISTRY: new disjoint ASCII-tagged derive_seed families RSRC_STREAM_BASE (0x0052_5352_4300_0000 "RSRC"), reuse CLIM_STREAM_BASE for light_cap, SPEC_STREAM_BASE for species assignment — zero SimRng draws, so introduction is spawn-draw-order-neutral (the soil R1.0/ADR-008 hash-neutral precedent), making each later coupling's draw-order change independently attributable.


## Extinction policy (supersedes ADR-005)

ADR-005 constant-population / NO-EXTINCTION is SUPERSEDED, not floored. Justification: the [0.5,1.5] strictly-positive band (soil.rs:191, climate.rs:119) and the 0.05 floor (lib.rs:209) existed for exactly ONE reason — to avoid zero-weight degeneracy in the cumulative WF sampler — and that sampler is DELETED, so the floors are obsolete by construction, not weakened. With a conserved-J balance, "harvest < maintenance" is a well-defined state and the only honest response is death.

POLICY: an organism dies when Energy <= 0 (starved, eaten, parasitized, or chem-poisoned); its residual J + Biomass return to the cell's detritus pool (conservation closes — death feeds decomposers, never leaks currency). A species sub-population CAN integrate to zero and the species vanishes — this is the DESIRED, informative emergent outcome (out-competed / prey-base collapsed / over-predated / self-poisoned), and the precondition for R3 species genuinely out-competing each other rather than each refilling a fixed Nₛ. The WHOLE WORLD can go extinct (every organism starves); the existing population==0 fast path (lib.rs:243) handles it; a self-extinguishing run is a valid, reproducible outcome, not a bug.

THE ONLY RETAINED BOUND IS A NON-FITNESS ALLOCATOR GUARD: MaxPopulation(u32), config-pinned, caps the entity Vec growth that emergent births create. Per the adversarial fix, the naive "skip births in ascending-OrgId order" is REJECTED because it is seed-correlated selection dressed as an allocator detail. Instead we pin ONE of two honest options (keystone Q for the human): (a) set MaxPopulation far above any equilibrium the resource budget can support, so it is genuinely never load-bearing, and TEST that it is never hit in the pinned config; OR (b) if it can bind, make over-cap birth refusal seed-NEUTRAL via a global RNG-drawn cull/birth-lottery on the existing stream, and DOCUMENT it as acknowledged selection. A skipped birth never consumes the parent's endowment (pinned). There is NO per-lineage floor.

This break lands behind its OWN deliberate re-pin + the 🛑 human sign-off it re-opens (ADR-005 explicitly protected this), with ledger_closes as the replacement safety invariant the constant-N guard used to provide.

## Invariant risks

- #3 (the real risk, per the adversarial verdict): determinism of the SIM is a property of iteration order + draw order + write visibility, NONE of which i64 arithmetic buys. CHEMOSTAT has ~7 ordered passes/tick plus mid-tick spawn/despawn; Bevy Query order is archetype/table order, NOT OrgId order (selection() sorts explicitly at lib.rs:246 precisely for this). MITIGATION (binding): every order-dependent pass operates on a pre-sorted (cell,SpeciesId,OrgId) index buffer, never raw Query order; archetype-reorder test guards it.
- #3: trophic in-place mutation needs two &mut to different entities from one Query — impossible naively in Bevy. MITIGATION: explicit EntityId index buffer + snapshot-of-prey-stores + ordered apply; contention/visibility semantics PINNED as a design decision (changes emergent dynamics, so it is a decision not an impl detail).
- #3: variable RNG draw count (births vary) makes the stream reproducible ONLY IF the entire causal chain upstream of every draw is bit-identical AND births are enumerated in a fixed canonical order. There is no 'prove the RNG later' — F3 is where it holds or collapses. MITIGATION: draw-count folded into the hash (fails fast); reproducibility harness gated BEFORE F3 merge; multi-ISA CI.
- #3: cap-saturation silently destroys quanta (saturating_add at a cap). MITIGATION: checked_add on ledger accounts routing overflow to a named third tap; ledger_closes asserts no saturation event and exact closure.
- #3: geometry LUTs derived from f64 at startup would diverge cross-platform. MITIGATION: LUTs baked as const array literals.
- #6: MaxPopulation 'skip births in OrgId order' is seed-correlated SELECTION dressed as an allocator guard. MITIGATION: set it non-load-bearing-and-tested, OR seed-neutral lottery + documented as acknowledged selection (keystone Q).
- #2: snapshot must stay read-only/off-hash and GDScript must compute no biology. RISK: new channels (pools/chem/flow) + GSS3 bump. MITIGATION: godot decoder updated same commit; snapshot proven RNG-free + hash-neutral (existing test pattern).
- #1: vector-DB sidecar must stay subprocess/never-linked; check_license.sh part-B currently keyed to literal 'oracle-slim'. MITIGATION: generalize part-B to cover relations-index (F6).
- #7: new pinned constants (caps, regen rates, shifts, LUTs, scales) proliferate — all must be pinned/recorded; cross-version reproducibility not guaranteed.
