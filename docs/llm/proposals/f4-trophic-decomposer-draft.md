## Winning design — THE OBLIGATE LOOP

Three couplings over `PoolStock`, all `i64`, RNG-free, ordered by `(cell_index, SpeciesId, OrgId)`. F4 DELETES
the `free_nutrient` arm of the F3 influx system (light-INFLUX stays — solar is the only true source) and makes
`free_nutrient` endogenous.

### 1. PLANTS DEPLETE + SHED

Autotroph uptake debits `PoolStock[cell].free_nutrient` via `Strategy.affinity[1]` (the GrowthRate→free_nutrient
anchor). Plants shed detritus on TWO arms — both paired pool↔org debit/credit (conserve J):
- **(a) death-deposit** — carcass→detritus on death (F3's existing carcass→detritus arm).
- **(b) litterfall** — a continuous fraction of the EXCRETE step routes inefficiency-carbon to
  `PoolStock[cell].detritus` every tick, so a living canopy rains litter even without death.

### 2. DECOMPOSER MINERALIZES (new `mineralize` branch)

Keyed on `strategy.role == Decomposer`. A decomposer:
1. taps `PoolStock[cell].detritus` via its now-meaningful `affinity[2]` slot;
2. demands against the **FROZEN start-of-tick snapshot** (F3's contention rule);
3. apportions co-located decomposers' shares via `fixed::apportion`;
4. SPLITS the granted joules through `split_budget(granted, strategy.budget)`:
   - the Maintenance/Defense fraction is **RESPIRED** (the decomposer's own living metabolism — this is what
     makes it a species, not a free converter);
   - the **MINERALIZATION** remainder is re-deposited into the SAME cell's `free_nutrient` pool.

The per-org mineralization fraction is a NEW `Strategy.mineralize_rate: u16` permille, expressed from the
AcetateOverflow/pta anchor (`gp.rs:42,217,367` — pta GO-8959 already documented as "the Layer-3
detritus/mineralization tap"), so the split is **gene-driven, not a constant**. This is a paired
detritus-debit / (free_nutrient-credit + RESPIRED-tap) move — conserves J exactly; `assert_ledger_closes`
holds every tick.

### 3. PLANTS RE-UPTAKE next tick → the cycle

OBLIGATE because, with the free_nutrient INFLUX arm deleted, the ONLY source of plant-available nutrient is
decomposer mineralization of shed detritus. CONTENTION uses F3's frozen start-of-tick snapshot:
- a carcass shed THIS tick feeds the decomposer NEXT tick;
- mineralized free_nutrient THIS tick feeds plants NEXT tick.

No within-tick feedback — and that lag IS the gameplay tell (a CRISPRi knockdown ripples to the plant
population a few ticks later). Kill the decomposer → detritus accumulates as a dead sink, free_nutrient drains
to 0, plants fall below `MAINTENANCE_FLOOR` → plant crash. Emergent niche separation: plants win in high-light
/high-free_nutrient cells, decomposers in high-detritus cells; disjoint `PoolStock` channels, no shared
nutritional niche.

### New fixed chain

F3's chain (`advance_tick → solar_influx → metabolism → maintenance → reproduce_or_die → measure →
assert_ledger_closes`) gains the `mineralize` system after `metabolism` (so decomposers tap the same
frozen-snapshot detritus plants/carcasses fed) and the litterfall arm folds into the existing EXCRETE step;
the `free_nutrient`-INFLUX arm is removed from `solar_influx`. The FlowMatrix is reset at the start of every
tick and accumulated during `trophic_transfer`/`mineralize`. All `.chain()` single-threaded (ADR-002).

## The FlowMatrix — MEASURED, the emergent inversion of the fabricated cosine

`FlowMatrix` = S×S net-integer J flows, MEASURED during `trophic_transfer`/`mineralize`. New
`crates/sim-core/src/relations.rs` (or co-located in `trophic.rs`):

```rust
pub struct FlowMatrix { s: usize, j: Vec<i64> }  // row-major; j[i*S + j] = NET J that flowed FROM species j INTO species i this generation
```

`s = SpeciesRegistry.entries.len()` (the registry is an ordered `Vec` indexed by `SpeciesId` — the indices ARE
registry ordinals, lib.rs:117). Built as a Bevy `Resource`, **reset to zero at the START of every tick**
(per-generation flow, not cumulative). All `i64`, no float, no cosine, no embedding.

### INVERSION of ADR-014

Verified against `rel-relations-draft.md:22-28`: the Rel draft built `A[i][j]` by "EXACT quantized cosine over
the `[u16;D]` genome vectors" — a FABRICATED input divorced from real interaction, collapsed to a `[0.5,1.5]`
factor fed INTO selection. **F4 makes A a MEASURED OUTPUT:** every conserved J transfer that crosses a species
boundary is RECORDED at the moment it happens, keyed by (source `SpeciesId`, dest `SpeciesId`).

Because detritus/free_nutrient pool quanta carry no per-quantum species tag (that would be unbounded), F4
attributes provenance via per-cell, per-window deposit ledgers (a clean integer mechanism): a
`DetritusDeposits: [i64; S]` and `NutrientMints: [i64; S]` per cell, reset each window. When a decomposer
harvests detritus, `flow[decomposer][plant] += harvested_J` apportioned over the species that deposited that
cell's detritus via `fixed::apportion`. When a plant takes up free_nutrient, `flow[plant][decomposer] +=
uptake_J` apportioned over who minted it. Future predation (OPT-B) slots in identically: prey→predator net J.

### CONSERVATION CONTRACT — `flow_matrix_rows_sum_to_zero` (the F4 analogue of `ledger_closes`)

The diagonal-pairing semantics (the more honest framing, a cleaner integer identity than an antisymmetric
form): EVERY credit `A[i][j] += x` (species i gained x attributable to j) carries a paired SELF-DEBIT
`A[i][i] -= x`, so:

> **`Σ_j A[i*S + j] == 0` BY CONSTRUCTION for every row i.**

A structural integer identity, asserted every tick via `flow_matrix_rows_sum_to_zero` (HARD under the
`determinism` feature, `debug_assert` otherwise). The diagonal is the per-species self-reference sink that
absorbs the row's net; the OFF-diagonals are the inter-species edges the relations VIEW renders.

Semantics: `A[i][j] > 0` = j is a net energy SOURCE for i. decomposer→plant via free_nutrient is the
mutualistic limb; **both off-diagonals positive when the loop runs, collapsing when the decomposer is knocked
out.** A flow that doesn't balance is a bug, not a re-pin.

### DETERMINISM + HASH

- Accumulated in the canonical `(cell, SpeciesId, OrgId)` sorted-Vec order F3 builds once per tick; the
  row-sum==0 pairing applied per-grant IN that fixed order; provenance via `fixed::apportion` (`u128`
  intermediates, identical on x86_64/aarch64 — rides the F3 multi-ISA gate). Integer add only, never `HashMap`
  (inv #3).
- **HASH-NEUTRAL at F4 introduction** — the soil-R1.0 / ledger / LiveTotal / Strategy precedent: a measurement
  derived from already-hashed pools/orgs adds nothing to the hash. The matrix rides the snapshot/observe path,
  NOT `hash_world`. It becomes hash-load-bearing ONLY if a later ADR couples it back into selection-demand;
  until then it does not itself force a re-pin (it rides ALONGSIDE F4.1's re-pin but is not its cause).
- EXPOSURE: `Simulation::flow_matrix() -> &FlowMatrix` (read-only accessor parallel to `species_strategy()`,
  lib.rs:805).

The vector-DB / cosine apparatus from the Rel draft SURVIVES but is **DEMOTED to a view-only analytics layer
OVER this measured matrix** (the firewall the Rel draft itself mandated, `rel-relations-draft.md:30`). The
fabricated-cosine `RelationMatrix`-into-selection is RETIRED. ADR-014's "cosine into selection" is marked

## Selection modifier — trophic state IS the J balance (no Wright-Fisher)

All three designs land the same reframe, binding: under F3/F4 "selection" is NO LONGER Wright-Fisher. F3
DELETES `fitness()`/`selection()` (lib.rs:289-303) and replaces them with energy-funded `reproduce_or_die`
(births from surplus J, deaths on starvation). So "trophic state feeds selection" is realized PHYSICALLY: an
organism's trophic success IS its J balance, which already decides who breeds and who starves.

The residual hook is a `ResourceModifier` trait (inv #5 seam, mirroring `EnvironmentModifier`/`ClimateModifier`),
feeding a strictly-positive integer-permille `[0.5,1.5]` factor that biases DEMAND (the F3 BLOCKER #1 rule:
scales demand PRE-apportion, NEVER the granted amount, NO f64 multiply on the i64 Energy/Biomass hash path).

```rust
trait ResourceModifier { fn demand_factor_permille(&self, role: TrophicRole, trophic_state: u16) -> u16; }
```

In-core default `LocalPoolMatchModifier`, returning an INTEGER permille clamped to `[500, 1500]` (1000 =
neutral). STRICTLY POSITIVE: floored at 500 so demand never zeroes (a starved org still dies via the honest
maintenance debit, not a zeroed weight), capped at 1500 so a perfectly-placed org can't run away. This
preserves F3's contention fairness and keeps the chemostat bounded-non-zero.

- `trophic_state` per org = the frozen-snapshot integer ratio of its role's primary channel availability to a
  reference: a Decomposer reads `PoolStock[cell].detritus` vs the static `ResourceField` detritus target; a
  plant reads `free_nutrient` vs target — quantized to permille via the audited `fixed::to_unit_u16` chokepoint.
  RNG-free.
- Applied PRE-apportion via integer scaling, so the single floored `granted` value is BOTH the pool debit AND
  the org credit (conserved, no mint). A plant in a free_nutrient-rich cell (decomposer working nearby) demands
  at factor>1000 and out-competes its cell-mates in `fixed::apportion` contention; a plant in a
  mineralization-starved cell demands at factor<1000 and loses. THAT is spatial selection driven by the trophic
  LOOP, not a static field.
- NEUTRAL-AT-DEFAULT discipline (ADR-012 E3 / soil / climate precedent): factor==1000 until trophic state
  actually diverges, so the re-pin captures the STRUCTURE, not a behavioral surprise.
- A Stage-5 LLM impl swaps behind the same trait without touching the metabolism arithmetic (inv #5).
  Multi-ISA stable (integer ratio + clamp, no transcendental, canonical-order walk).

The FlowMatrix is explicitly NOT read into this modifier at F4 (keeping the modifier a pure function of
`(role, local pool)` keeps the FlowMatrix hash-neutral and the demand path simple). Coupling FlowMatrix→demand
is a deliberate LATER ADR with its own re-pin.
