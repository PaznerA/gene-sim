## Winning design (the synthesis)

### Metabolism — `uptake → convert → excrete`, pure `i64`, ZERO `SimRng` draws

The keystone determinism win all three architectures converge on: metabolism draws nothing.

- **Pool representation.** Add a SEPARATE mutable `PoolStock{ light: Vec<i64>, free_nutrient: Vec<i64>,
  detritus: Vec<i64>, width, height }`, seeded ONCE at reset by quantizing the static `ResourceField` `[0,1]`
  through the single audited `f64→int` chokepoint (`fixed::to_unit_u16(v) as i64 * CELL_J_SCALE`). The static
  `ResourceField` (f32) survives UNTOUCHED as (a) the render-snapshot source and (b) the per-cell
  carrying-cap / influx target — preserving F1's byte-exact off-stream field and the GSS render path.
  `assert!(WORLD_DIMS == RESOURCE_DIMS)` at reset (both 32×32) so `Position` maps 1:1 to a `PoolStock` cell via
  `cell_index = pos.y*WORLD_DIMS.0 + pos.x` with NO resample.
- **The pipeline**, per org in canonical `(cell_index, SpeciesId, OrgId)` sorted order, reading `PoolStock` +
  the species' cached `Strategy`:
  1. **UPTAKE** — Monod-like integer saturation, NO float/RNG: for each channel `r` the org's `role` taps
     (Autotroph→light; Heterotroph→free_nutrient/detritus; Decomposer→detritus), `demand_r` = integer fn of
     `affinity[r]` × Biomass × `budget[Acquisition]`; `uptake_r = (Vmax*S_r)/(K_half+S_r)` in `u128` floored to
     `i64`, where `S_r` = the FROZEN-snapshot stock. Granted leaves `PoolStock`, enters the org.
  2. **CONVERT** — `fixed::split_budget(granted, strategy.budget)` splits joules across the 5 channels:
     Acquisition re-invests, Growth→Biomass, Reproduction→Energy reserve (the birth fund), Maintenance/Defense
     →respiration debit. Trophic-efficiency loss (`EFF_NUM/EFF_DEN < 1`) is RESPIRED.
  3. **MAINTENANCE** — per-org upkeep debit funded by `budget[Maintenance]`, subtracted from Energy → RESPIRED.
     This is what makes starvation possible. Runs BEFORE reproduce (only survivors breed).
  4. **EXCRETE** — trophic-inefficiency carbon + an overflow fraction → `PoolStock[cell].detritus` (the F4
     decomposer hook); any uptake exceeding `BIOMASS_CAP/ENERGY_CAP` → `Ledger.overflow` (never silent
     saturating-sub).
- **Two-pool organism** (kept from Arch0): `Energy(i64)` = free reserve, `Biomass(i64)` = structural mass —
  so uptake scales with body size and death returns a biomass quantum to detritus. Every division goes through
  `fixed::apportion`/`split_budget` so totals conserve exactly.

### Lifecycle — energy-funded `reproduce_or_die` REPLACES constant-N Wright-Fisher

- **Order within the tick:** maintenance BEFORE reproduce — only survivors breed; a dying org doesn't draw a
  birth (pins the RNG draw set cleanly).
- **DEATH (runs first, RNG-FREE):** starvation (Energy < `MAINTENANCE_FLOOR` after the maintenance debit) and
  senescence (`Age(u32)` ≥ `AGE_MAX`, HARD ceiling at F3; soft age→maintenance coupling deferred). A dying
  org's residual Biomass+Energy deposits to `PoolStock[cell].detritus` (carcass→detritus, conserving J);
  despawn via a COLLECTED `Vec<Entity>` (never mutate-during-query).
- **BIRTH (runs second — the ONLY SimRng consumer in the whole tick):** Energy ≥ `REPRO_THRESHOLD` → parent
  SPENDS `OFFSPRING_ENDOWMENT` (`parent.Energy -= endowment`, `child.Energy = endowment − seed_biomass`,
  `child.Biomass = seed_biomass` — CONSERVED, no minting; the child seed-Biomass comes OUT of the endowment).
  Child inherits Species (off-stream), Genotype/DroughtTol/ThermalTol with mutation, Position = parent cell +
  a Moore dispersal step. **Per-birth draw shape is a FIXED compile-time constant: D mutation draws + 1
  dispersal draw, UNCONDITIONAL** (always the same count regardless of outcome) so a `DrawCount` mismatch
  localizes the bug. A skipped/over-cap birth does NOT consume the endowment (else the allocator leaks J); the
  cap check PRECEDES the endowment debit, the draws happen regardless (draw-count independent of cap state).
  Child OrgId from a monotonic `NextOrgId` (NEVER reused slot indices). Widen OrgId to `u64` (population is now
  unbounded).
- **Population = ∫(births − deaths).** Per-species identity persists via the `Species` tag + per-species
  cached Strategy; a sub-population can integrate to zero and vanish (desired extinction). `target_pop` demoted
  to INITIAL spawn count. `Observation.population_size` becomes per-species.
- **Allocator guard:** a config-pinned `MaxPopulation(u32)` set FAR above any resource-supportable
  equilibrium, with a test asserting it is NEVER hit in the pinned config (provably non-load-bearing — avoids
  the inv-#6 "skip births in OrgId order" hidden-selection trap).

### Contention — FROZEN SNAPSHOT + APPORTION (the one real disagreement, resolved)

The load-bearing decision: clone `PoolStock` at tick start; all orgs compute DEMAND against the frozen value;
a deterministic second pass apportions each cell's actual available J across co-located demanders via
`fixed::apportion(available, &demands)` (largest-remainder, ties to lowest index), decrementing the pool ONCE.

This beats immediate-apply because OrgId is birth-order/seed-derived — with energy-funded births,
who-ate-first compounds into who-survived-to-breed, so an immediate-apply positional bias IS a hidden
selection gradient correlated with OrgId (the class inv #6 forbids). Frozen+apportion makes scarcity bite a
crowded cell as a unit (the legible boom→crash), is order-independent, and is **already the human-APPROVED
default** (DECISIONS.md). The SAME frozen rule covers carcass→detritus: deaths this tick deposit, consumers
this tick read the FROZEN pre-death detritus → a carcass feeds NEXT tick (clean causal ordering, no
within-tick feedback loop). Cost: an `O(cells × 3)` per-tick clone (1024×3 i64 — trivial).

### Ordering — ONE canonical order, built once per tick

Every order-dependent pass collects live orgs into a Vec and `sort_unstable_by_key(|r| (cell_index,
SpeciesId, OrgId))` BEFORE iterating — never Query/archetype order, never HashMap (inv #3). cell FIRST
(contention is per-cell). This single key orders metabolism demand-gathering, contention apportionment,
maintenance, reproduce_or_die, AND the carcass→detritus death-deposit + overflow-routing pass. The canonical
vector is built ONCE over the LIVING set at tick start and NOT rebuilt after within-tick deaths (deaths only
MARK entities for despawn, applied after reproduce).

New fixed chain (replaces lib.rs:627): `advance_tick → solar_influx → metabolism(uptake/convert/excrete) →
maintenance → reproduce_or_die → measure → assert_ledger_closes`, all `.chain()` single-threaded (ADR-002).

### Ledger closure — asserted EVERY tick (semantic gate > bit-hash)

`Σ(all live J) == initial_total + influx − respired − overflow`, asserted every tick. Live total =
`Σ(PoolStock light+free_nutrient+detritus)` + `Σ(per-org Energy)` + `Σ(per-org Biomass)`. chem = 0 until F5
(documented zero). The THREE named taps, each written by EXACTLY ONE place: **INFLUX** (the `solar_influx`
system, first in the chain, mints solar J into `PoolStock.light` per cell up to the static `ResourceField.light`
cap; the ONLY source of new J; free_nutrient regen toward its target is treated as INFLUX at F3 — a
documented F3→F4 open tap, closed endogenously by the plant→detritus→decomposer→free_nutrient loop at F4);
**RESPIRED** (metabolism CONVERT efficiency loss + MAINTENANCE debit + the un-excreted dead-org residual — the
only sink); **OVERFLOW** (cap-saturation only — pool ceilings, org Energy/Biomass caps, detritus cap; saturating
logic ROUTES the spill, never silently clamps). The infra landed now: `live_total` is modelled by the
`LiveTotal` helper, `assert_ledger_closes` runs LAST under the `determinism` feature as a HARD assert.

### Draw-count stability + the `DrawCount` fail-fast

With N free, the per-tick draw count is data-dependent — FINE for determinism PROVIDED the draw order is a
pure function of the sorted org list. RULE: draws ONLY in reproduce_or_die, ONLY for orgs passing the energy
threshold, in canonical order, a FIXED shape per birth (D mutation + 1 dispersal, UNCONDITIONAL). ZERO draws
in influx/metabolism/maintenance/measure. Add a `DrawCount(u64)` resource incremented at every `next_u64`,
FOLDED INTO `hash_world` alongside `final_word` — an off-by-one in birth enumeration then breaks the hash
LOCALLY on one ISA instead of silently drifting into a plausible-but-wrong reproducible value. Pair with a
`same_seed_same_population_trajectory` test asserting two runs produce identical per-tick population vectors.

## Adversarial findings folded in as BINDING contracts

All three adversarial passes returned **NOT READY as written**, with a consensus 2 blockers + 5 secondary
fixes. These are binding on the gated births/deaths slice:

1. **BLOCKER — the harvest-multiplier graft reintroduces float on the hash path.** Grafting
   `DroughtTol/ThermalTol` soil+climate `fitness_factor` (f64, soil.rs:188/climate.rs:114) as MULTIPLIERS on
   i64 uptake puts `i64 × f64 → truncate-to-i64` into hashed Energy/Biomass every tick — the #1 x86_64/aarch64
   divergence vector (fp-contraction of the abs/add chain). **FIX (binding):** re-express the match factor as
   an INTEGER permille — `fixed::to_unit_u16(factor)` ONCE → scale demand via `fixed::apportion` — so NO f64
   multiply touches live Energy/Biomass. The match factor scales **DEMAND (pre-apportion)**, never the granted
   amount (post-apportion), so the single floored granted value is BOTH the pool debit and the org credit (no
   mint/destroy). This preserves the ADR-011/012 spatial selection as a REAL integer energetic advantage.
2. **BLOCKER — the multi-ISA CI gate must exist and be GREEN on TODAY's hash FIRST.** Landed in this slice
   (above). It is the precondition, not a follow-up.
3. **Heritable f64 fields stay f64 on the hash.** `Genotype/DroughtTol/ThermalTol.to_bits()` and the
   `allele_freq` f64 mean remain f64-folded after F3 — the "determinism as a property of i64" thesis is NOT
   fully delivered at F3. **Contract:** F3 is EXPLICITLY scoped as "pools/energy/biomass are i64; the heritable
   trait fields stay f64 and the multi-ISA gate must PROVE those f64 fields are byte-stable across arches"
   (the `-fp-contract=off` + `target-cpu=generic` pin is the guard). i64-izing the traits + allele_freq is a
   SEPARATE later re-pin, not smuggled into F3.
4. **Mutation draw schedule UNCONDITIONAL.** Today D=0 (verbatim copy). F3 adds mutation → D becomes a NEW
   draw count; pin D as a compile-time constant, EVERY threshold-passing org draws EXACTLY D+1 words
   regardless of outcome, else `DrawCount` only catches the bug AFTER divergence on whichever ISA runs second.
   Dispersal uses `next_u64()%range` integer-only (no `unit_f64`, so no float in the hashed Position).
5. **Death-deposit + overflow-routing in canonical order.** Integer addition is commutative so the detritus
   SUM is order-independent, BUT which carcass's deposit OVERFLOWS (when a detritus cap is hit mid-accumulation)
   is order-dependent — so the death-deposit + overflow pass MUST run in canonical `(cell,SpeciesId,OrgId)`
   order, not Query order.
6. **Apportion index = canonical order.** `fixed::apportion` breaks ties to the LOWEST index; that index must
   be the position in a demand vector BUILT in canonical order, or the leftover quantum silently shifts who
   breeds. Test with a deliberately over-subscribed cell so leftover-quantum assignment is exercised + pinned.
7. **No bare `clamp`/`min`/`saturating_*` in the J path.** `saturating_sub` silently floors at 0 → destroys a
   deficit. Maintenance respires `min(debit, Energy)` to RESPIRED and the shortfall triggers death; the
   tap fields are mutated only via audited single-writer helpers. CONVERT efficiency `respired = granted −
   Σ(kept)` computed as a RESIDUAL (not an independent `granted*EFF` divide that double-floors a quantum).
   Influx into a full cell: `influx += actually_added` (post-clamp), the spill → overflow.
8. **Carcass-deposit + despawn atomic vs the `live_total` read.** `assert_ledger_closes` runs LAST, AFTER all
   deposits AND despawns complete in maintenance/reproduce — so a carcass's J is counted in detritus OR in the
   org, never both, never neither.
9. **Pin the f32→i64 PoolStock seed.** Add a test pinning the EXACT quantized i64 `PoolStock` for the default
   seed (it sets `initial_total`; any drift silently shifts the whole ledger baseline). The f32→f64→u16→i64
   widen is exact/deterministic, but the audit claim must be the pinned test, not a comment.
10. **i64 numeric budget.** Release builds WRAP on overflow (same on both arches, so not an ISA-divergence
    risk — but a silent ledger violation the assert catches). Pin a budget so `Σ(pools+Energy+Biomass)` over
    `MaxPopulation` orgs + 1024 cells × 3 channels × `POOL_CAP` + influx accumulation across the pinned
    generation count stays `< i64::MAX` with margin. `apportion`'s `u128` intermediates are arch-safe.
