# ADR-013 — Multi-species ecosystem (roadmap R3): SpeciesRegistry + per-species Wright-Fisher, per-species snapshot planes, species-routed edits

> **DRAFT — generated by the `r3-multispecies-design` workflow (scheduled run). DESIGN PROPOSAL ONLY — awaiting human sign-off. R3 is a stop-the-line invariant gate (inv #6); no code written.**

> ⚠️ The independent judge phase failed (schema bug, since fixed in the workflow def); the architecture pick comes from the synthesize agent's own trade-off reasoning over the understand+design maps, not an independent judge panel.

## Context

R3 (docs/llm/TASKS.md) requires multiple species competing across regions. Today the core is strictly single-species: ONE shared ECS World holds one flat panmictic population; one `GenomeRes(Genome)` (lib.rs:83) + one `BaseGrowthRate(f64)` (lib.rs:89) drive everything; `selection()` (lib.rs:218) is one Wright-Fisher pool over all entities (2 draws/offspring, 2N draws/gen) with no species partition; organisms carry no species tag (OrgId/Energy/Genotype/DroughtTol/ThermalTol/Position only); snapshot (lib.rs:464) + GSS2 wire format (snapshot.rs:37, CHANNEL_COUNT=6) emit one set of 3 organism channels; edits (`with_genome_and_rng` lib.rs:555, `apply_edit_region` lib.rs:588, harness `EditAction` lib.rs:78, godot-sim apply_edit lib.rs:205) all target the one implicit genome with no species selector; the UI specimen log (main.gd:70) is a single flat stream.

This touches three stop-the-line invariants: #2 (all genotype->phenotype biology stays in sim-core/genome; renderer read-only), #3 (single seeded ChaCha8Rng threaded explicitly, no HashMap iteration in sim logic, no transcendentals, PINNED determinism hash 0x9fad_2c9f_d298_f73a in determinism_hash_is_pinned at lib.rs:739 with the ADR-011 re-pin ledger procedure), and #6 (agency at species/region granularity, never per-organism). Per the LOOP, R3 is a stop-the-line invariant gate: this ADR is a DESIGN PROPOSAL ONLY and must get human sign-off before any implementation slice runs. Three architectures were explored (shared-world fixed-size partition; isolated sub-sims coupled by per-cell carrying capacity K; SpeciesRegistry with density-coupled sub-populations).


## Decision

Adopt a single shared World with every organism carrying a heritable, off-stream-assigned `Species(SpeciesId)` tag, backed by an ordered `SpeciesRegistry(Vec<SpeciesEntry{genome, base_growth, name}>)` resource indexed positionally by SpeciesId (0..S = registry index; NEVER a HashMap). `selection()` becomes S independent Wright-Fisher pools iterated in ascending SpeciesId order; offspring inherit their parent's species, so cross-species breeding is structurally impossible. Snapshot bumps GSS2->GSS3 and emits per-species organism channel planes (density/allele_freq/fitness) plus the 3 GLOBAL soil channels once. Every edit gains a mandatory SpeciesId selector and (for region edits) a Species filter on the org query. The crispr gate (`evaluate_region_edit`/`apply_edit`, crispr/lib.rs:620) is unchanged because it already takes `genome: &Genome` by reference (clean seam, inv #5). The UI gains one PAGE per species, each a TABLE of that species' incremental specimen log (baseline tree + one tree per successful edit), all renderer-only (inv #2).

The competition mechanic is sequenced into TWO re-pins, not one: R3 SHIPS FIRST with FIXED per-species sub-population sizes (each pool refills to its own Nₛ — smallest determinism delta, preserves the constant-population / no-extinction guarantee of ADR-005), and density-coupled variable sub-population sizes (the lever that lets a fitter/edited species actually outcompete and shift relative abundance) lands as a SEPARATE later slice because it changes draw semantics and re-opens the extinction policy.


## Chosen approach

Winner: "Species Registry + Per-Species Sub-Population Wright-Fisher" (Design 3) as the structural spine — an ordered Vec-indexed SpeciesRegistry is the cleanest inv-#3-honoring home for per-species genome + cached base_growth, mirrors the genome crate's existing Vec+newtype-id pattern (LocusId), and the off-stream Species tag keeps the per-organism spawn draw order (g0,energy,drought,thermal) byte-unchanged so the re-pin stays small and interpretable.

Grafts from the runners-up: (a) From Design 1 — adopt its FIXED-size sub-population posture as the FIRST competition step so the initial re-pin captures only the structural folds (Species in hash + registry param-count fold + partition reordering) with the 2N draw COUNT preserved, deferring variable abundance. (b) From Design 2 — when variable abundance does land, prefer its per-cell carrying-capacity K(cell) seam (integer ratio math, no transcendentals) as the cleanest, most testable competition coupling, over Design 3's global Σ-fitness resize, because K localizes competition to where species actually co-occur and keeps the rounding/tie-break auditable. (c) From all three — keep snapshot RNG-free/read-only (the proven cheap part), bump GSS3 magic so stale readers fail loud, normalize density per-species (each by its own busiest cell) so sparse species stay visible, and use `#[serde(default)]` on the new EditAction species field so old actions.ndjson journals replay as species 0.

Rejected: Design 2 as the spine (loses the constant-TOTAL-population guarantee immediately and front-loads the K rounding/tie-break determinism hotspot before the structural plumbing is even in). Design 3's density-coupling in the FIRST slice (changes per-gen genotypes + needs an OrgId free-list for shrinking/growing pools — too much determinism surface for one re-pin). Merging species into one Vec<Locus> (would mis-map WeightedSumMap's flat-index weights, gp.rs:81 — keep one Genome == one species).


## Consequences

POSITIVE: per-organism spawn draw order unchanged (Species off-stream like placement); crispr gate untouched (inv #5); snapshot stays provably zero-RNG/read-only (existing read-only tests carry over); edits stay species+region scoped (inv #6); old journals/saves stay loadable via serde defaults + single-species default roster; one re-pin per slice keeps each ledger entry tight and auditable.

NEGATIVE / COSTS: a determinism RE-PIN is unavoidable and happens TWICE across R3 (once for structural multi-species, once for variable-abundance coupling) — each re-pins the single literal at lib.rs:739 in the SAME commit with a new ledger line (lib.rs:728-733) per ADR-011 consequences. The GSS3 wire-format break + struct rewrite cascades through snapshot.rs, godot-sim, snapshot.gd and the whole main.gd render/specimen path in lockstep. WeightedSumMap weights are keyed to global flat param indices (gp.rs:81), so genuinely DIFFERENT per-species genome layouts are BLOCKED until weights become locus/ontology-keyed (explicit follow-up, not in R3). Per-species density normalization makes cross-species abundance non-comparable from the overlay alone (needs the per-species population readout to disambiguate). With fixed sub-populations shipping first, species cannot yet shift relative abundance until the second slice — R3's headline mechanic is staged.

FOLLOW-UPS spun out: weights keyed off OntologyTags/locus ids (unblocks distinct species genomes); consuming the currently-dead traits (Reflectance/DroughtTolerance/Fecundity, gp.rs) to differentiate species niches; per-cell (not global) climate coupling (already flagged in selection()).

## Re-pin plan

One re-pin per slice, two re-pins total across R3, sequenced so each ledger entry is interpretable. R3-A: NO re-pin (data-model plumbing only; default 1-species roster + off-stream Species tag keeps the hash byte-identical at 0x9fad_2c9f_d298_f73a; this is the deliberate guard that the plumbing is hash-neutral). R3-B: FIRST re-pin — driver is the partitioned per-species selection plus folding Species, the per-species genome param-count sum, and species_count into hash_world; the 2N draw COUNT is preserved so the re-pin captures ONLY the structural reordering + new hashed fields; update the literal at lib.rs:739 and append a ledger line at lib.rs:728-733 citing ADR-013 R3-B, in the same commit. R3-C/R3-D/R3-E: NO re-pin (snapshot stays off the hash path; edit-routing is <=1 draw and RNG-neutral; UI is render-only). R3-F: SECOND re-pin — driver is density-coupled variable sub-population sizes (changes per-gen genotypes/draw semantics); re-pin the literal again in the same commit with its own ledger line and a fresh ADR/addendum recording the K rounding + tie-break rule. Each re-pin follows the ADR-011 consequences procedure exactly; never re-pin and change unrelated logic in the same commit.


## Invariant risks

- #3 (determinism): TWO deliberate re-pins (R3-B, R3-F). Risk of a silent wrong-literal re-pin — mitigate by shipping the default single-species roster so R3-A is provably hash-neutral and R3-B's delta is attributable to named structural folds only. Every per-species store MUST be a Vec indexed by contiguous SpeciesId; every loop iterates ascending SpeciesId then OrgId; NO HashMap may enter sim logic (the per-species genome registry, base-growth, and snapshot cell buffers are the high-risk spots). K-coupling in R3-F must use integer ratio math only (no transcendentals) with a fully specified tie-break, or species balance becomes platform-fragile.
- #3 (RNG order): Species must be assigned off-stream (loop index at spawn, like placement) so the per-org 4-draw order (g0,energy,drought,thermal) is byte-unchanged; per-species selection must keep exactly 2 draws/offspring and visit species in ascending SpeciesId order over the (species,id)-sorted parent Vec, or the stream diverges.
- #2 (biology in core only): per-species phenotypes/observe must be computed in the core and only DISPLAYED by GDScript; the per-species specimen pages, vitals, and de-dup logic in main.gd must read core-exported phenotypes and never compute genotype->phenotype. Risk that the per-species UI tempts biology into GDScript — forbid.
- #6 (species/region granularity): edits must carry a SpeciesId selector and region edits must filter the org query by species==sp; never expose a per-organism edit handle. The brush stays region+species scoped.
- #5 (pluggable science): the crispr gate and the soil/climate modifiers must stay untouched — the gate already takes &Genome by reference; only callers add the selector. Risk if K-coupling (R3-F) leaks competition logic into a modifier instead of selection().
- WeightedSumMap weight-mapping (gp.rs:81): keyed to global flat param indices — only identical per-species genome templates are safe in R3. Distinct species genome layouts would silently mis-map trait weights; this is BLOCKED until weights are re-keyed (follow-up), and the ADR must say so explicitly.
- Wire/journal compatibility: GSS3 magic bump + EditAction serde field + SeedJson/run_id roster changes must fail loud on stale formats (GSS3 magic) yet load old journals via #[serde(default)]; a half-updated reader silently mis-parsing is the risk.
